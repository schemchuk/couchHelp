---

## Мова спілкування

- Завжди спілкуйся українською мовою
- Технічні терміни (назви функцій, змінних, команди) — англійською
- Коментарі в коді — українською
- Повідомлення комітів — українською, чіткі і конкретні:
  - ✅ `додано базову схему бази даних`
  - ❌ `fix`, `update`, `changes`

---

## Git workflow

- Після кожного логічно завершеного блоку → коміт → пуш. Без винятків.
- Гілки: `main` — продакшн, `dev` — розробка, `feature/назва-фічі` — нові функції

---

## Принципи написання коду

- Кожне рішення має витримати масштабування
- Ніколи не довіряй вхідним даним без валідації
- Ніколи не хардкодь ключі — тільки через змінні середовища
- `.env.example` завжди актуальний і синхронізований
- Якщо є підозра, що API-ключ або токен недійсний — попередь користувача перш ніж вносити зміни

## Нові модулі — правило INSTRUCTIONS.md

Перед реалізацією будь-якого нового модуля або агента:

1. Створи папку модуля
2. Створи `INSTRUCTIONS.md` з розділами: призначення, стек, правила поведінки, інтеграції, змінні середовища, приклади
3. Закоміть: `додано INSTRUCTIONS.md для [назва модуля]`
4. Запушити на GitHub
5. Тільки після цього — починай реалізацію
